# Prompt Anywhere 调试与测试指南

## 环境准备

### 1. 安装依赖

```bash
cd Prompt-Anywhere
pip install -r requirements.txt
```

依赖列表：
- PySide6 >= 6.7
- pynput >= 1.7.6
- markdown >= 3.5
- Pygments >= 2.16
- PyYAML >= 6.0.1
- python-dotenv >= 1.0.0

### 2. 配置文件

项目使用多层配置（优先级从高到低）：
1. 环境变量（`.env` 文件）
2. `config.yaml`
3. 代码默认值

修改 `.env` 或 `config.yaml` 即可调整应用行为，无需改代码。

### 手动 UI 测试（最终验收流程）

按以下步骤验证 MVP 功能：

1. **启动应用**
   ```bash
   python -m app.main
   ```
   预期：窗口置顶显示，标题为 "Prompt Anywhere"

2. **测试全局快捷键**
   - 按 `Ctrl + Alt + P`，窗口应隐藏
   - 再按一次，窗口应显示并置顶，搜索框自动聚焦

3. **测试系统托盘**
   - 点击窗口关闭按钮，应隐藏到托盘（不是退出）
   - 右键托盘图标，选择"显示 / 隐藏"，窗口应切换显示状态
   - 选择"打开数据目录"，应打开 `./data` 文件夹
   - 选择"退出"，应用应完全退出

4. **新建分类**
   - 点击左侧分类栏的 `+` 按钮
   - 输入分类名："Coding"
   - 预期：`data/Coding/` 目录被创建

5. **新建提示词**
   - 点击中间提示词列表的 `+` 按钮
   - 填写：
     - 名称：代码精简
     - 分类：Coding（默认）
     - 格式：.md
     - 内容：输入 Markdown 文本
   - 预期：`data/Coding/代码精简.md` 文件被创建

6. **编辑与保存**
   - 在编辑模式修改内容
   - 标题栏应显示 `*` 标记表示未保存
   - 按 `Ctrl + S` 或点击"保存"按钮
   - 预期：文件被保存，`*` 标记消失

7. **渲染模式**
   - 点击"渲染模式"按钮
   - 预期：Markdown 被渲染为排版后的 HTML（含语法高亮）

8. **搜索**
   - 在顶部搜索框输入"精简"
   - 预期：中间列表只显示匹配的文件

9. **复制**
   - 选中一个提示词，点击"复制"
   - 状态栏显示"已复制到剪贴板"
   - 粘贴到 ChatGPT/Claude 网页版，内容为 Markdown 原文（非富文本）

10. **导出**
    - 选中提示词，点击"导出"
    - 选择保存路径
    - 预期：文件原样复制到目标路径

11. **删除提示词**
    - 选中提示词，点击"删除"
    - 预期：弹出确认对话框，确认后文件被删除

12. **删除分类**
    - 右键分类名，选择"删除"
    - 预期：弹出确认对话框，确认后整个分类目录被删除

13. **重启验证**
    - 关闭并重新启动应用
    - 预期：之前的分类和提示词仍然存在

## 常见问题排查

### 全局快捷键不工作

1. 检查是否有其他应用占用了 `Ctrl + Alt + P`
2. 尝试修改 `.env` 中的 `GLOBAL_HOTKEY`
3. Windows 上，在 UAC 弹窗/任务管理器等 elevated 窗口中，快捷键需要管理员权限

### 窗口置顶失效

- 确保 PySide6 >= 6.7
- 全屏游戏/视频可能覆盖置顶窗口（Windows DWM 策略）

### 托盘图标残留

- Windows Explorer 的已知 bug，鼠标悬停在托盘区域即可刷新
- 应用正常退出时调用 `tray.hide()`

### Markdown 渲染异常

- 检查 `python-markdown` 和 `Pygments` 是否正确安装
- 查看控制台是否有异常日志

### 文件编码问题

- 确保 `.env` 中 `FILE_ENCODING=utf-8`
- Windows 上避免使用 GBK 编码的文件

#### 模糊搜索验证

```python

# 精确匹配
r = search_matcher.fuzzy_match("代码", "代码审查")
print(r.matched, r.score)  # True, 80+

# 错别字容忍
r = search_matcher.fuzzy_match("代码审", "代码审查")
print(r.matched, r.score)  # True, 60+

# 拼音匹配
r = search_matcher.match_pinyin("dmsc", "daimashencha", "dmsc")

```

#### 搜索性能测试

```python
import time
from app.services.search_service import search_service

search_service.rebuild_index()
start = time.time()
for _ in range(10):
    search_id, worker = search_service.search_async('代码')
    worker.start()
    worker.wait()
elapsed = time.time() - start
print(f'10 searches in {elapsed*1000:.1f}ms')
```

## 常见问题排查

### 拼音搜索不生效

1. 检查 `.env` 中 `SEARCH_ENABLE_PINYIN=true`
2. 检查 `pypinyin` 是否安装：`pip show pypinyin`
3. 检查日志中是否有 pinyin_service 相关错误

### 模糊搜索结果不准确

调整 `.env` 中的配置：
```
SEARCH_FUZZY_MODE=strict    # 更严格
SEARCH_FUZZY_MODE=balanced  # 平衡（默认）
SEARCH_FUZZY_MODE=loose     # 更宽松
```

### 语义搜索API报错

1. 检查 API Base URL 格式（末尾是否有 `/v1`）
2. 检查 API Key 是否有效
3. 测试网络连通性
4. 查看日志中的 EmbeddingService 错误

### AI模板助手API报错

1. 检查 `AI_TEMPLATE_ENABLED=true`
2. 检查 Base URL、API Key、Model 配置
3. 尝试增加超时时间：`AI_TEMPLATE_TIMEOUT_SECONDS=60`
4. 查看日志中的 AITemplateService 错误

### 知识库元数据不同步

from app.services.knowledge_base_service import knowledge_base_service
knowledge_base_service.full_sync()
