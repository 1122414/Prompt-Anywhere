# Prompt Anywhere 调试与测试指南

## 环境准备

### 1. 安装依赖

```bash
cd Prompt-Anywhere
pip install -r requirements.txt
```

依赖列表：
- PySide6 >= 6.7
- pynput >= 1.7.6
- markdown >= 3.5
- Pygments >= 2.16
- PyYAML >= 6.0.1
- python-dotenv >= 1.0.0

### 2. 配置文件

项目使用多层配置（优先级从高到低）：
1. 环境变量（`.env` 文件）
2. `config.yaml`
3. 代码默认值

修改 `.env` 或 `config.yaml` 即可调整应用行为，无需改代码。

## 功能测试

### 自动化测试（服务层）

```bash
python tests/test_functional.py
```

测试覆盖：
- 文件服务（分类/提示词 CRUD）
- 搜索服务（关键词匹配）
- Markdown 渲染（HTML 生成）
- 单例模式验证

### 手动 UI 测试（最终验收流程）

按以下步骤验证 MVP 功能：

1. **启动应用**
   ```bash
   python -m app.main
   ```
   预期：窗口置顶显示，标题为 "Prompt Anywhere"

2. **测试全局快捷键**
   - 按 `Ctrl + Alt + P`，窗口应隐藏
   - 再按一次，窗口应显示并置顶，搜索框自动聚焦

3. **测试系统托盘**
   - 点击窗口关闭按钮，应隐藏到托盘（不是退出）
   - 右键托盘图标，选择"显示 / 隐藏"，窗口应切换显示状态
   - 选择"打开数据目录"，应打开 `./data` 文件夹
   - 选择"退出"，应用应完全退出

4. **新建分类**
   - 点击左侧分类栏的 `+` 按钮
   - 输入分类名："Coding"
   - 预期：`data/Coding/` 目录被创建

5. **新建提示词**
   - 点击中间提示词列表的 `+` 按钮
   - 填写：
     - 名称：代码精简
     - 分类：Coding（默认）
     - 格式：.md
     - 内容：输入 Markdown 文本
   - 预期：`data/Coding/代码精简.md` 文件被创建

6. **编辑与保存**
   - 在编辑模式修改内容
   - 标题栏应显示 `*` 标记表示未保存
   - 按 `Ctrl + S` 或点击"保存"按钮
   - 预期：文件被保存，`*` 标记消失

7. **渲染模式**
   - 点击"渲染模式"按钮
   - 预期：Markdown 被渲染为排版后的 HTML（含语法高亮）

8. **搜索**
   - 在顶部搜索框输入"精简"
   - 预期：中间列表只显示匹配的文件

9. **复制**
   - 选中一个提示词，点击"复制"
   - 状态栏显示"已复制到剪贴板"
   - 粘贴到 ChatGPT/Claude 网页版，内容为 Markdown 原文（非富文本）

10. **导出**
    - 选中提示词，点击"导出"
    - 选择保存路径
    - 预期：文件原样复制到目标路径

11. **删除提示词**
    - 选中提示词，点击"删除"
    - 预期：弹出确认对话框，确认后文件被删除

12. **删除分类**
    - 右键分类名，选择"删除"
    - 预期：弹出确认对话框，确认后整个分类目录被删除

13. **重启验证**
    - 关闭并重新启动应用
    - 预期：之前的分类和提示词仍然存在

## 常见问题排查

### 全局快捷键不工作

1. 检查是否有其他应用占用了 `Ctrl + Alt + P`
2. 尝试修改 `.env` 中的 `GLOBAL_HOTKEY`
3. Windows 上，在 UAC 弹窗/任务管理器等 elevated 窗口中，快捷键需要管理员权限

### 窗口置顶失效

- 确保 PySide6 >= 6.7
- 全屏游戏/视频可能覆盖置顶窗口（Windows DWM 策略）

### 托盘图标残留

- Windows Explorer 的已知 bug，鼠标悬停在托盘区域即可刷新
- 应用正常退出时调用 `tray.hide()`

### Markdown 渲染异常

- 检查 `python-markdown` 和 `Pygments` 是否正确安装
- 查看控制台是否有异常日志

### 文件编码问题

- 确保 `.env` 中 `FILE_ENCODING=utf-8`
- Windows 上避免使用 GBK 编码的文件

## 项目结构

```
Prompt-Anywhere/
├── app/
│   ├── main.py              # 应用入口
│   ├── config.py            # 配置管理
│   ├── constants.py         # 常量定义
│   ├── ui/
│   │   ├── main_window.py   # 主窗口（整合所有面板）
│   │   ├── panels.py        # 三栏面板（分类/提示词/编辑器）
│   │   ├── dialogs.py       # 对话框（新建分类/提示词）
│   │   └── tray.py          # 系统托盘
│   ├── services/
│   │   ├── file_service.py  # 文件读写服务
│   │   ├── search_service.py # 搜索服务
│   │   ├── clipboard_service.py # 剪贴板服务
│   │   └── export_service.py # 导出服务
│   └── utils/
│       ├── markdown_utils.py # Markdown 渲染
│       └── syntax_highlighter.py # Markdown 语法高亮
├── tests/
│   └── test_functional.py   # 功能测试脚本
├── data/                    # 提示词数据目录
├── exports/                 # 导出目录
├── .env                     # 环境变量
├── config.yaml              # 配置文件
└── requirements.txt         # 依赖列表
```
