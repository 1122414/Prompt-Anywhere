# Prompt Anywhere MVP Plan

## 1. MVP 定位

Prompt Anywhere 的 MVP 版本只做一个目标：

> 在电脑任何地方，通过一个可置顶显示的本地桌面工具，快速保存、查找、编辑、渲染、复制、导出自然语言提示词。

第一版不做智能能力、不做后端服务、不做数据库、不做结构化 Prompt 管理系统。

核心原则：

- 用户写什么，就保存什么。
- 用户看到的是普通 `.md` / `.txt` 文件。
- 用户复制的是原始自然语言内容。
- 分类就是本地文件夹。
- 提示词就是本地 Markdown / 文本文件。
- Markdown 文件需要支持编辑模式和渲染模式。

---

## 2. MVP 必须实现的功能

### 2.1 桌面窗口能力

必须实现：

1. PySide6 >= 6.7 本地桌面应用（6.7+ 规避系统托盘已知 bug）。
2. 窗口默认置顶显示。
3. 窗口可以拖动、缩放。
4. 支持隐藏到系统托盘。
5. 支持从系统托盘重新打开。
6. 支持退出应用。
7. 支持全局快捷键呼出 / 隐藏窗口。

推荐默认快捷键：

```txt
Ctrl + Alt + P
```

快捷键行为：

```txt
如果窗口隐藏，则显示并置顶。
如果窗口已显示，则隐藏。
显示窗口后，自动聚焦搜索框。
```

## 3. 本地文件存储设计

MVP 不使用数据库。

直接使用本地文件夹作为数据源。

```txt
prompt-anywhere/
├── app/
├── data/
│   ├── Coding/
│   │   ├── 代码审查.md
│   │   └── 代码精简.md
│   ├── 面试/
│   │   ├── 项目介绍.md
│   │   └── HR反问.md
│   ├── 简历/
│   │   └── 项目润色.md
│   └── 日常/
│       └── 总结归纳.md
├── exports/
├── config.yaml
└── README.md
```

规则：

```txt
分类 = data 目录下的一级文件夹
提示词 = 分类文件夹下的 .md / .txt 文件
正文 = 文件里的原始自然语言内容
```

示例：

```txt
data/Coding/代码精简.md
```

内容：

```md
你是一名资深代码重构专家。请帮我检查下面代码，优先删除重复逻辑、合并无用分支、减少复杂度，不要引入新的架构和过度抽象。
```

---

## 4. 文件格式要求

MVP 支持两种文件格式：

```txt
.md
.txt
```

其中：

- `.md` 文件支持 Markdown 编辑和 Markdown 渲染。
- `.txt` 文件只按纯文本显示和编辑。
- 复制时始终复制文件原文。
- 导出时始终原样导出文件内容。

不需要支持：

```txt
YAML frontmatter
metadata
PromptItem
type
tags
description
AGENTS 类型
Skill 类型
模板类型
```

## 6. 分类功能

分类就是文件夹。

MVP 必须支持：

1. 新建分类。
2. 重命名分类。
3. 删除分类。
4. 点击分类后显示该分类下的提示词文件。
5. "全部"分类显示所有分类下的提示词文件。
6. 空状态处理：
   - 首次启动时自动创建 `data/` 目录。
   - `data/` 目录为空时，左侧分类列表显示"全部"，中间提示词列表显示空状态提示（如"暂无提示词，点击 + 新建"）。
   - 分类下无提示词文件时，显示该分类的空状态提示。

分类规则：

```txt
新建分类 = 在 data/ 下创建文件夹
重命名分类 = 重命名文件夹
删除分类 = 删除文件夹
```

删除分类前必须二次确认。

## 7. 提示词文件功能

提示词就是 `.md` 或 `.txt` 文件。

MVP 必须支持：

1. 新建提示词。
2. 编辑提示词。
3. 保存提示词。
4. 删除提示词（删除前需二次确认）。
5. 重命名提示词。
6. 复制提示词全文。
7. 导出提示词文件。
8. 点击提示词后加载文件内容。

新建提示词时，用户只需要填写：

```txt
文件名
所属分类（默认选中当前左侧选中的分类）
文件格式：.md / .txt
正文内容
```

保存规则：

```txt
data/分类名/文件名.md
data/分类名/文件名.txt
```

## 8. Markdown 编辑 / 渲染双模式

这是 MVP 必须实现的核心体验。

### 8.1 编辑模式

编辑模式用于修改提示词原文。

要求：

1. 使用可编辑文本框显示原始 Markdown。
2. 用户可以直接修改内容。
3. 点击保存后写回本地文件。
4. 未保存时需要有状态提示（如标题栏或状态栏显示"已修改"标记）。
5. 切换文件前，如果当前内容未保存，弹出模态确认对话框，提供"保存"、"不保存"、"取消"三个选项。

编辑模式显示的是原始内容，例如：

```md
# 代码审查提示词

你是一名资深代码审查专家。

请帮我检查下面代码：

1. 只指出关键问题
2. 不要过度重构
3. 给出最小修改建议
```

---

### 8.2 渲染模式

渲染模式用于阅读 Markdown。

要求：

1. `.md` 文件在渲染模式下显示为排版后的 Markdown。
2. 支持标题、列表、引用、代码块、粗体、斜体等常见 Markdown 样式。
3. 渲染模式下内容不可直接编辑。
4. 用户可以一键切回编辑模式。
5. `.txt` 文件可以禁用渲染模式，或以纯文本方式展示。

渲染模式展示效果应类似：

```txt
代码审查提示词

你是一名资深代码审查专家。

请帮我检查下面代码：

1. 只指出关键问题
2. 不要过度重构
3. 给出最小修改建议
```

---

### 8.3 复制规则

无论当前处于编辑模式还是渲染模式：

```txt
点击复制时，始终复制文件原始内容。
```

也就是说，复制的是 Markdown 源码，而不是渲染后的富文本。

复制时必须仅写入纯文本格式（`text/plain`），避免携带富文本格式导致粘贴到网页时出现异常。

## 9. 搜索功能

MVP 搜索只做普通关键词匹配。

搜索范围：

1. 文件名。
2. 文件正文内容。

搜索结果：

1. 显示匹配到的提示词文件。
2. 点击结果后打开对应文件。
3. 搜索为空时恢复正常分类列表。

不做：

```txt
语义搜索
向量检索
AI 推荐
自动分类
```

## 10. 复制功能

MVP 必须支持一键复制。

复制内容：

```txt
当前选中的提示词文件原文（纯文本格式，不含富文本样式）
```

复制后显示提示：

```txt
已复制到剪贴板
```

复制目标场景：

```txt
ChatGPT 网页版
Claude 网页版
Gemini 网页版
opencode
Claude Code
Codex
任意聊天窗口
```

## 11. 导出功能

MVP 必须支持导出当前提示词文件。

导出规则：

1. 用户选择一个提示词文件。
2. 点击导出。
3. 选择导出路径。
4. 原样复制当前文件到目标路径。

支持导出格式：

```txt
.md
.txt
```

不需要格式转换。

## 12. 系统托盘功能

系统托盘右键菜单必须包含：

```txt
显示 / 隐藏
新建提示词
打开数据目录
退出
```

其中：

- “显示 / 隐藏”用于切换主窗口。
- “新建提示词”打开新建提示词窗口。

## 14. 推荐技术栈

```txt
Python
PySide6 >= 6.7
pynput
python-markdown
Pygments
PyYAML
```

用途：

```txt
PySide6 >= 6.7：桌面窗口、置顶、托盘、UI（6.7+ 规避托盘 hide/show 已知 bug）
pynput：全局快捷键（唯一活跃维护的跨平台方案）
python-markdown：将 .md 内容转成 HTML（支持 GFM 扩展）
Pygments：代码块语法高亮
PyYAML：读取配置
```

Markdown 渲染方案：

```txt
编辑模式：QPlainTextEdit + QSyntaxHighlighter（Markdown 语法高亮）
渲染模式：python-markdown → Pygments 高亮 → HTML → QTextBrowser.setHtml()
```

不推荐使用 QWebEngineView，原因：
- 打包体积增加约 200MB
- 打包复杂度高（需处理 QtWebEngineProcess）
- MVP 不需要 Mermaid / 数学公式等高级渲染

QTextBrowser + python-markdown + Pygments 方案：
- 体积增量仅约 5MB
- 支持完整 GFM 语法（表格、任务列表、删除线等）
- 支持代码块语法高亮（500+ 语言）
- 打包零复杂度

## 16. MVP 开发阶段与验收标准

### Phase 1：桌面壳子

实现：

1. PySide6 主窗口。
2. 默认置顶。
3. 托盘图标。
4. 托盘显示 / 隐藏。
5. 全局快捷键显示 / 隐藏。

验收：

```txt
启动后出现置顶窗口。
关闭窗口时可以隐藏到托盘。
托盘可以重新打开窗口。
Ctrl + Alt + P 可以呼出 / 隐藏窗口。
```

---

### Phase 2：本地文件读取

实现：

1. 读取 `data/` 目录。
2. 将一级文件夹显示为分类。
3. 将 `.md` / `.txt` 文件显示为提示词。
4. 点击文件后读取正文。

验收：

```txt
左侧能看到分类。
中间能看到提示词文件。
点击提示词后右侧能看到文件内容。
```

---

### Phase 3：新建 / 编辑 / 保存 / 删除

实现：

1. 新建分类。
2. 重命名分类。
3. 删除分类。
4. 新建提示词。
5. 重命名提示词。
6. 编辑提示词正文。
7. 保存提示词正文。
8. 删除提示词。

验收：

```txt
用户可以在 UI 中完成分类和提示词的增删改。
所有修改都会同步到本地 data 文件夹。
重启应用后内容不丢失。
```

---

### Phase 4：Markdown 双模式

实现：

1. 编辑模式。
2. 渲染模式。
3. 模式切换按钮。
4. `.md` 文件支持 Markdown 渲染。
5. `.txt` 文件支持纯文本展示。
6. 复制时始终复制原文。

验收：

```txt
.md 文件可以在编辑模式查看源码。
.md 文件可以在渲染模式查看排版效果。
两种模式可以自由切换。
编辑后保存，渲染结果同步更新。
点击复制时复制 Markdown 原文。
```

---

### Phase 5：搜索 / 复制 / 导出

实现：

1. 文件名搜索。
2. 正文内容搜索。
3. 一键复制当前提示词原文。
4. 导出当前提示词文件。

验收：

```txt
搜索关键词可以找到对应提示词。
点击复制后可以粘贴到 ChatGPT 网页版。
点击导出后能得到原样 .md / .txt 文件。
```

## 17. MVP 最终验收流程

MVP 完成后，必须能跑通下面流程：

```txt
1. 启动 Prompt Anywhere。
2. 窗口置顶显示。
3. 按 Ctrl + Alt + P 隐藏窗口。
4. 再按 Ctrl + Alt + P 呼出窗口。
5. 新建分类：Coding。
6. 新建提示词：代码精简.md。
7. 输入自然语言 Markdown 提示词。
8. 点击保存。
9. 切换到渲染模式，看到 Markdown 渲染效果。
10. 切换回编辑模式，继续修改正文。
11. 搜索“精简”，找到该提示词。
12. 点击复制。
13. 粘贴到 ChatGPT 网页版，内容为 Markdown 原文。
14. 点击导出，导出为 .md 文件。
15. 重启应用，分类和提示词仍然存在。
```

## 19. 一句话总结

Prompt Anywhere MVP 是：

> 一个 PySide6 本地桌面置顶提示词文件夹工具。

底层规则：

```txt
分类 = 文件夹
提示词 = .md / .txt 文件
正文 = 用户自然语言
编辑模式 = 修改原文
渲染模式 = 查看 Markdown 排版

```
